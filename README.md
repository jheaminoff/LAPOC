# LAPOC — LA City Planning One-Stop-Shop

A proof-of-concept agentic chatbot that lets Los Angeles residents, developers, and contractors look up permit status, planning case details, and process workflows — grounded in real LA City planning processes and data structures.

The assistant auto-detects whether you're a homeowner, developer, or contractor and adapts its language accordingly.

**Stack:** Python FastAPI + SQLite (backend) · React 18 + TypeScript + Vite (frontend) · Azure OpenAI (LLM) · Azure Speech (voice)

---

## Quick Start

```bash
# 1. Bootstrap the project (one command)
bash init

# 2. Edit backend/.env with your Azure OpenAI credentials
#    (see Environment Variables section below)

# 3. Start both servers — backend first, then frontend
cd backend && uvicorn main:app --reload &   # http://localhost:8000
cd frontend && npm run dev                  # http://localhost:5173
```

| **Case detail** | Full status, fees paid/owed, next action, hearing date, portal link for any LADBS permit or City Planning case |

| **Dynamic suggestions** | Context-aware follow-up questions generated by LLM after each query |

## API Reference

### `POST /chat` — Main agent endpoint

Accepts a conversation history and returns the agent's reply.

**Request:**
```json
{
  "persona": "auto",
  "messages": [
    { "role": "user", "content": "What's the status of 2815 Sunset Blvd?" }
  ]
}
```

| Field | Type | Description |
|---|---|---|
| `persona` | `string` | `"auto"` (detect from chat), `"resident"`, `"developer"`, or `"contractor"` |
| `messages` | `array` | Full conversation history in `[{role, content}]` format |

**Response:**
```json
{
  "reply": "PARCEL: 2815 Sunset Blvd\n  APN: 5462-001-016\n  ...",
  "speech_text": "I found a parcel at 2815 Sunset Boulevard...",
  "tool_calls_made": ["lookup_parcel"],
  "detected_persona": "resident",
  "suggestions": ["What permits are on this property?", "Can I build an ADU here?"]
}
```

| Field | Description |
|---|---|
| `reply` | Agent's text reply (may contain structured `PARCEL:`, `CASE DETAIL:`, `WORKFLOW:`, or `MAP:` blocks) |
| `speech_text` | Condensed 1-2 sentence spoken version for TTS |
| `tool_calls_made` | List of tool names the agent invoked |
| `detected_persona` | Detected role (`resident`/`developer`/`contractor`) or `null` |
| `suggestions` | 4 context-aware suggested follow-up questions |

### `GET /plots/{apn}` — Parcel lookup by APN
### `GET /plots/search/address?q={query}` — Address search
### `GET /cases/{case_id}` — Case detail
### `GET /cases/random` — Random case ID (for demo suggestions)
### `GET /speech-token` — Azure Speech STS token (proxied)
### `GET /health` — Health check `{"status": "ok"}`

## Agent System

### Persona system prompts

Jinja2 blocks in `agent/prompts/system_prompts.jinja` control how the LLM speaks:

| Persona | Tone | Use Case |
|---|---|---|
| `resident` | Plain language, empathetic, define jargon | Homeowners asking about their property |
| `developer` | Technical LA planning terms, flag risks | Project teams, entitlement questions |
| `contractor` | LADBS terminology, status/inspection focus | Licensed contractors, plan check |
| `auto` | Detects role from user language | Adapts on first response |

### Agent tools (5)

| Tool | Source | Returns |
|---|---|---|
| `lookup_parcel(apn_or_address)` | SQLite | `PARCEL:` block with property + cases |
| `get_case_detail(case_id)` | SQLite | `CASE DETAIL:` block with status, fees, action |
| `get_workflow(process_type, persona)` | SQLite | `WORKFLOW:` block with step timeline + guidance |
| `lookup_address(address)` | BOE GeoQuery API | Prose + `MAP:` block with coordinates + jurisdictions |
| `check_adu_eligibility(apn_or_address)` | SQLite → ZIMAS API | `ADU ELIGIBILITY CHECK:` block |

The agent loop (`run_agent` in `agent.py`) runs up to 6 iterations with `tool_choice: "auto"`. Each iteration calls the LLM, executes any tool calls, feeds results back, and repeats until the LLM produces a final text response.

### Structured text protocol

When a tool returns data, the LLM passes it through verbatim in the response using sentinel prefixes. The frontend parses these and renders visual cards:

| Sentinel | Frontend component | Renders as |
|---|---|---|
| `PARCEL:` | `ParcelCard` | Property info + cases table |
| `CASE DETAIL:` | `CaseDetailCard` | Status banner + fees + next action |
| `WORKFLOW:` | `WorkflowTimeline` | Animated step timeline |
| `MAP:` | `MapCard` | Leaflet map with marker |
| `ADU ELIGIBILITY CHECK:` | (raw prose) | Plain text in bubble |

## Seed Data

The database ships with synthetic data modelled on real LA processes:

- **10 parcels** — verified real APNs from LA County GIS, covering Silver Lake, Hollywood, Venice, DTLA, West Adams, South LA, Leimert Park, Sunset Strip
- **30 cases** — realistic permit/planning case numbers in proper LADBS and City Planning formats with varied statuses (Issued, Corrections Needed, Appeal Pending, LOD Issued, PC on Hold, etc.)
- **135 workflow steps** — 20 process types including ADU (standard + custom), CUB, Zone Change, TOC, CDP, EIR, Grading, Electrical, Plumbing, HVAC, Fire Sprinkler, and more
- **99 persona guidance rows** — plain-language (resident), technical (developer), and LADBS-focused (contractor) guidance per step

Reseed (destructive):
```bash
cd backend && python seed/seed_data.py
```

**Data note:** This is a POC with synthetic case data — permit numbers, applicant names, fees, and statuses are fabricated for demonstration. Parcel APNs and zoning data are real. For production, replace with live LADBS PermitLA or City Planning API.

## Development

### Commands

| What | Where | Command |
|---|---|---|
| Backend dev server | `backend/` | `uvicorn main:app --reload` |
| Reseed DB | `backend/` | `python seed/seed_data.py` |
| Frontend dev server | `frontend/` | `npm run dev` |

| Lint | `frontend/` | `npx eslint src/` |
| Prod build | `frontend/` | `npm run build` (`tsc -b && vite build`) |
| API docs (Swagger) | — | `open http://localhost:8000/docs` |

**Verification order:** lint → typecheck → build.

### Frontend conventions

- **CSS Modules only** — no Tailwind, no inline styles. Each component gets a paired `*.module.css`.
- **`@/` alias** → `src/`. Use for all non-relative imports.
- **`VITE_API_URL`**: empty string in dev (Vite proxy); `https://tre-app-lapoc.azurewebsites.net` in prod.
- **tsconfig strict**: `strict`, `noUnusedLocals`, `noUnusedParameters` on. Prefix unused params with `_`.
- **Azure Speech SDK** — must be in `optimizeDeps.include` in `vite.config.ts`.
- **`presentation.html`** — second rollup entry for demo mode.

### Backend gotchas

- `load_dotenv()` is called **before** all imports in `main.py:5-7`. Any module-level code reading env vars must be there or after startup.
- Azure OpenAI endpoint: no trailing slash, no `/openai/...` suffix.
- DB location: `lapoc.db` relative to CWD (`.gitignore`). CI reseeds every deploy.
- `/cases/random` route must be declared before `/{case_id}` (already correct).
- CORS origins hardcoded in `main.py` — add new deploy URLs there.
- BOE GeoQuery API key is hardcoded in `tools.py:262` (`B5thBZ3BXlR1waoUvderfnrMETLwk2SE`).

### Frontend gotchas

- **30s fetch timeout** — `Chat.tsx` aborts `/chat` after 30 seconds. Deep agent chains can hit this.
- **Greeting audio cached in IndexedDB** — key `"greeting"` via `src/utils/audioCache.ts`. Stale audio survives reloads; clear IndexedDB manually when testing.
- **WorkflowTimeline collapse** — >7 steps shows first 5 with "Show all N steps" toggle.
- Vite proxy routes `/plots`, `/cases`, `/chat`, `/health`, `/speech-token`, `/static` → `localhost:8000`.

`azure-pipelines-frontend.yml` — triggered on any `main` push:

2. `npm ci && npm run build`

## Known Limitations

- **Synthetic data only** — no live LADBS or City Planning API connection
- **Azure OpenAI required** — not configured for standard OpenAI or other providers
- **No authentication** — API is completely open
- **English only** — no Spanish / multilingual support
- **No persistence** — conversation history not saved between sessions
- **30s timeout** — deep 6-iteration agent chains can time out
